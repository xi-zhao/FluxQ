<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| doctor findings -> CI verdict | Workspace and dependency findings are reclassified into blocking versus advisory outcomes for machine consumers. |
| CLI flag -> runtime policy evaluation | `--ci` changes the contract seen by JSON and JSONL consumers. |
| doctor report -> exit-code mapping | CI relies on shell semantics matching the structured verdict. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | T | doctor blocking/advisory classification | mitigate | Reuse the existing `issues` versus `advisories` split and only project it into explicit `blocking_issues` and `advisory_issues` fields. |
| T-04-03-02 | R | doctor CI decision payload | mitigate | Emit one shared policy envelope with explicit reason-code prefixes so JSON, JSONL, and exit codes all describe the same blocking state. |
| T-04-03-03 | D | shell-level CI behavior | mitigate | Map `verdict.status == "fail"` to exit code `2` and `pass` to `0` only when `--ci` is active, preserving legacy behavior otherwise. |
</threat_model>
