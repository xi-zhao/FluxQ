<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compare result -> persisted compare JSON | The in-memory compare decision becomes durable CI evidence under `.quantum/compare/`. |
| benchmark input resolution -> benchmark history JSON | A detached report or historical revision crosses into a persisted benchmark artifact that later plans will trust. |
| policy helper -> command payloads | Later commands will reuse one helper for machine-facing gate output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | T | `runtime/compare.py` persistence | mitigate | Serialize compare artifacts through `ensure_schema_payload()` so disk artifacts always keep `schema_version` and cannot silently drift from CLI JSON shape. |
| T-04-01-02 | S/T | `diagnostics/benchmark.py` imported-revision persistence | mitigate | Thread `source_kind` and `source_revision` from import resolution through benchmark persistence so a detached or historical benchmark cannot masquerade as the mutable current revision. |
| T-04-01-03 | R | `runtime/policy.py` shared envelope helper | mitigate | Centralize envelope assembly and next-action mapping so benchmark and doctor do not emit divergent reason-code or gate structures for the same policy concepts. |
</threat_model>
