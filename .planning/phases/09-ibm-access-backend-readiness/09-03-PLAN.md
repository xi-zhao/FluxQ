<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workspace IBM profile → backend discovery | 本地配置驱动对外部 provider metadata 的读取 |
| provider backend metadata → CLI JSON payload | 外部 target 状态被投影为用户/agent 决策依据 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | T | `backend_list.py` readiness projection | mitigate | 输出显式 `instance`, `reason_codes`, `readiness.status`，而不是单个推荐 backend，避免 target 选择被隐藏。 |
| T-09-03-02 | I | backend list payload | accept | backend 名称、队列数、qubit 数本身是 provider metadata；不输出 token、saved-account secret 或 raw headers。 |
| T-09-03-03 | E | backend auto-selection | mitigate | 明确禁止 `least_busy()` / `recommended_backend` / `selected_backend`，能力字段标注 `remote_submit=False`。 |
</threat_model>
