<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executor -> workspace commit files | Runtime execution crosses from in-memory results into shared mutable workspace state. |
| trace writer -> append-only event logs | Event streams must remain readable even when multiple processes target the same workspace. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | T | `runtime/executor.py` revision promotion | mitigate | Hold one workspace lease across revision promotion so current aliases cannot be interleaved between competing writers. |
| T-03-03-02 | D | `workspace/trace.py` event append path | mitigate | Append event records only while the workspace lease is held and fsync after writes so partial log records do not corrupt the event streams. |
| T-03-03-03 | R | interrupted-write recovery | mitigate | Detect leftover temp artifacts before commit and raise the structured recovery-required signal instead of silently proceeding with ambiguous state. |
</threat_model>
