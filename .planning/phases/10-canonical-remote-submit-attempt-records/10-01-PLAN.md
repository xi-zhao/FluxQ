## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local workspace manifest -> remote attempt store | durable submit metadata is written near authoritative local runtime state |
| provider receipt payload -> persisted JSON | untrusted provider metadata crosses into long-lived local artifacts |
| concurrent remote submits -> attempt id counter | multiple callers can contend for one workspace-local attempt namespace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | I | `provider_receipt.json` | mitigate | Recursively redact `token`, `auth`, `authorization`, `Authorization`, `api_key`, and nested `headers` before any write; lock this with runtime tests. |
| T-10-01-02 | T | `workspace.json` and active aliases | mitigate | Load `WorkspaceManifest` before and after persistence, assert `current_revision`, `active_spec`, and `active_report` are unchanged, and keep all attempt files under `.quantum/remote/attempts/`. |
| T-10-01-03 | D | attempt-id allocation | mitigate | Reserve `att_000001`-style ids under the workspace lock and write the counter through same-directory atomic replacement. |
