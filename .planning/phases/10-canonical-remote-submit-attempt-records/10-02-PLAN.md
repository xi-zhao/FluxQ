<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| canonical local input -> provider submit call | FluxQ turns trusted local intent / `QSpec` state into an external IBM job submission |
| provider job handle -> local attempt record | The first durable remote fact crosses back into the local workspace as a first-class control-plane object |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | T | backend / instance targeting | mitigate | Require explicit `--backend` and preserve explicit instance semantics from Phase 09; forbid `least_busy()` and implicit active-instance fallback. |
| T-10-02-02 | D | provider submit path | mitigate | Use the existing IBM access seam and fail closed before provider submission when access is unresolved or backend lookup fails. |
| T-10-02-03 | I | attempt persistence | mitigate | Persist only non-secret submit metadata and canonical hashes; never store token bytes or provider auth headers. |
| T-10-02-04 | R | local provenance | mitigate | Persist the local attempt immediately with canonical snapshots and provider job ID so the job can be re-opened later without ambiguity. |
</threat_model>
