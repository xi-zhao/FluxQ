## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| submit orchestration -> machine-readable JSON / JSONL | Internal provider and workspace state is projected into agent-facing contracts |
| blocked submit path -> CI / agent decision | Failure vocabulary must be stable and actionable without leaking sensitive details |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | I | JSON / JSONL payloads | mitigate | Restrict output to reason codes, next actions, attempt metadata, and non-secret provider descriptors; test redaction explicitly. |
| T-10-03-02 | R | submit lifecycle events | mitigate | Emit `submit_started`, `submit_persisted`, and `submit_completed` so event consumers can distinguish provider call, local persistence, and terminal submit status. |
| T-10-03-03 | D | blocked submit handling | mitigate | Use fail-closed reason codes and gate blocks for access / backend / provider errors instead of ambiguous partial-success wording. |
| T-10-03-04 | T | machine contract drift | mitigate | Reuse shared remediation and next-action helpers rather than duplicating remote-submit-only mappings inside the CLI. |
</threat_model>
