## Decisions Made

- Used a standard-library `ThreadingHTTPServer` seam so the gateway stays lightweight and does not introduce a second service framework for an intentionally thin boundary.
- Chose a JSON allowlist format keyed by `wechat_user.id` with a server-owned `workspace_key`, because the trust boundary requires the execution root to come from operator-managed state rather than caller input.
- Returned `launcher_request` alongside the thin response envelope so the gateway can remain transport-focused while later plans wire the real launcher and confirmation flow.
