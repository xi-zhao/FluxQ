<threat_model>
High-severity threats in this plan are blocking. Do not complete the plan until command-injection and cross-user state risks are mitigated in code and tests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| launcher request -> `claw` subprocess | Gateway payloads become local agent subprocess calls on the execution host |
| `claw` tool call -> `qrun` wrapper | Agent tool usage crosses into the FluxQ control plane |
| conversation state -> future turns | Persisted transcript data influences later agent behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-02-01 | T/E | `fluxq-qrun` | mitigate | Build argv lists without a shell, whitelist exact command families, and reject unknown verbs or raw shell fragments. |
| T-999.1-02-02 | E | `run-claw-agent` | mitigate | Require the gateway to call only `run-claw-agent`; keep direct `claw` invocation details inside the launcher seam. |
| T-999.1-02-03 | I/T | conversation transcripts | mitigate | Scope transcript files to `<workspace_key>/<conversation_id>`, cap retained history, and never reuse one user's transcript for another user. |
| T-999.1-02-04 | R/T | high-risk tool calls | mitigate | Emit `confirmation_required` with the exact approved request and summary metadata before any `remote submit` or spend-creating execution occurs. |
</threat_model>
