## Decisions Made

- Used one structured JSON request shape for the router so `claw` stays on a stable tool interface while FluxQ remains the execution authority.
- Stored pending approved tool requests inside the transcript state so a later confirmation step can execute the exact reviewed action instead of re-running prompt interpretation.
- Prefixed PATH with `integrations/cc-connect/bin` inside the launcher so confirmed tool execution can call `fluxq-qrun` directly without hardcoding absolute paths into the gateway.
