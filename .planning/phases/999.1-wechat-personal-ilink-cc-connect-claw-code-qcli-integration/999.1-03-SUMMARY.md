## Decisions Made

- Used a dedicated confirmation store under the gateway state root so confirmation identity, expiry, and one-time-use semantics stay outside transient chat memory.
- Replayed the previously approved tool request on confirm instead of re-parsing the user’s natural-language message, preserving the reviewed action across the trust boundary.
- Treated the real split-host smoke as a blocking human verification gate; the summary records the explicit approval and does not fabricate extra live execution details.
