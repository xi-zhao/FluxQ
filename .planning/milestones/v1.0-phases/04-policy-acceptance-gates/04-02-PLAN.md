<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| saved workspace baseline -> current revision compare result | CI trusts FluxQ to compare the approved baseline state against the mutable current revision without wrapper logic. |
| in-memory compare result -> `.quantum/compare/*.json` | Persisted compare artifacts become durable machine evidence for later policy composition. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | T | baseline/current compare policy activation | mitigate | Keep policy activation explicit in `compare_command()` through `--baseline`, `--expect`, and `--fail-on` flags rather than hidden `QSpec` hints. |
| T-04-02-02 | T/R | persisted compare JSON | mitigate | Serialize compare artifacts through `ensure_schema_payload()` so disk artifacts cannot silently drift from the CLI JSON contract. |
</threat_model>
