<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| saved baseline benchmark history -> current benchmark policy decision | CI relies on persisted benchmark evidence instead of ad hoc shell math. |
| CLI threshold flags -> runtime policy evaluator | User-supplied thresholds directly influence accept/reject behavior. |
| imported report/revision benchmark -> persisted benchmark history | Historical or detached benchmark inputs must not masquerade as the mutable current revision. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | S/T | baseline benchmark evidence loading | mitigate | Load the saved baseline benchmark file explicitly from `benchmarks/history/<baseline_revision>.json` and fail with `baseline_benchmark_missing` if it is absent. |
| T-04-03-02 | T | benchmark policy evaluation | mitigate | Check subject parity, backend presence, and comparability before threshold math, then emit stable benchmark-specific reason codes. |
| T-04-03-03 | R | imported-revision benchmark persistence | mitigate | Persist `source_kind` and `source_revision` on the report and use that revision for history naming instead of the mutable current manifest revision. |
</threat_model>
