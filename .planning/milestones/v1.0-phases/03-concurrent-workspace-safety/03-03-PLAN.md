<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executor -> workspace commit files | Runtime execution crosses from in-memory results into shared mutable workspace state for one reserved revision. |
| exec artifact writers -> current aliases | Generated code, synthesis results, and figure outputs cross into mutable alias files that later runs can overwrite. |
| trace writer -> append-only event logs | Event streams must remain readable even when multiple processes target the same workspace. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | T | `runtime/executor.py` revision promotion | mitigate | Hold one workspace lease from revision reservation through final promotion so latest files and current aliases cannot be interleaved between competing writers. |
| T-03-03-02 | D | `workspace/trace.py` event append path | mitigate | Append event records only while the workspace lease is held and fsync after writes so partial log records do not corrupt the event streams. |
| T-03-03-03 | T | artifact and figure alias writers | mitigate | Stage generated current aliases first and promote them atomically inside the guarded exec flow so failed runs cannot leave mixed code or figure aliases. |
| T-03-03-04 | R | interrupted-write recovery | mitigate | Detect leftover temp artifacts before commit and raise the structured recovery-required signal instead of silently proceeding with ambiguous state. |
</threat_model>
