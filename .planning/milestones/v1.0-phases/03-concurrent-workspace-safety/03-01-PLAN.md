<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process -> `.quantum/.workspace.lock` | Untrusted concurrent processes compete to bootstrap or mutate shared workspace state. |
| temp file -> bootstrap and current alias files | Interrupted or partial writes can appear beside authoritative workspace files such as `workspace.json`, `qrun.toml`, `events.jsonl`, `trace/events.ndjson`, and current aliases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | T | `workspace/manager.py` bootstrap writes | mitigate | Guard `load_or_init()` with the workspace lease and replace direct bootstrap writes with same-directory temp write plus fsync plus atomic replace so partial seed files never become authoritative. |
| T-03-01-02 | D | `workspace/locking.py` lease file | mitigate | Use exclusive lease acquisition with deterministic conflict metadata so concurrent writers fail fast instead of racing indefinitely. |
| T-03-01-03 | R | lock-holder identity | mitigate | Persist holder metadata in the lease file so later conflict payloads can attribute the active writer and support retry logic. |
| T-03-01-04 | T | `workspace/manifest.py` save path | mitigate | Move manifest persistence onto the atomic bootstrap helper so concurrent init or revision bumps cannot partially overwrite `workspace.json`. |
</threat_model>
