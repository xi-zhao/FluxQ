<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| verified bundle -> target workspace history | Foreign bundle bytes cross into a workspace that downstream commands will trust as local revision history. |
| imported history -> promoted current aliases | Imported immutable artifacts become the active workspace pointers after promotion. |
| target workspace existing state -> imported revision | Pre-existing target files may conflict with the bundle revision and must not be silently merged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | S/T | bundle import entrypoint | mitigate | Verify the bundle first and reject `pack_bundle_invalid` inputs before writing any target-workspace files. |
| T-05-03-02 | R | imported history and alias promotion | mitigate | Write immutable history first, then promote aliases and workspace manifest only from the imported revision artifacts. |
| T-05-03-03 | D | target workspace conflict / partial import | mitigate | Hold the workspace lock during import and fail with `pack_revision_conflict` on conflicting existing revision bytes. |
</threat_model>
