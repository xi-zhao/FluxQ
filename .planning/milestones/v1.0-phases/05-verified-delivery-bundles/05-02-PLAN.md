<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| copied pack root -> inspection verdict | Arbitrary filesystem bytes cross into a trust decision without any source-workspace guarantees. |
| bundle-local manifest -> digest verifier | The manifest itself becomes the authoritative map of what must be present and what bytes are trusted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | S/T | external bundle verification | mitigate | Verify bundles from bundle-local relative paths and digests instead of trusting source-workspace absolute paths. |
| T-05-02-02 | T | copied bundle bytes | mitigate | Fail closed on missing required entries or `bundle_digest_mismatch:<relative_path>` when any bundled file is tampered after packaging. |
| T-05-02-03 | R | machine-readable inspection output | mitigate | Emit stable `reason_codes`, `next_actions`, and `gate` fields so downstream agents do not reverse-engineer trust from prose. |
</threat_model>
