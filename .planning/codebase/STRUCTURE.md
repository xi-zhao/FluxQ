## Directory Purposes

**`src/`:**
- Purpose: Source root for the Python package and local build metadata.
- Contains: `src/quantum_runtime/`, `src/quantum_runtime.egg-info/`, and many empty sibling directories such as `src/assistant/`, `src/bootstrap/`, `src/cli/`, `src/services/`, and `src/tools/` that do not currently hold live FluxQ Python source files.
- Key files: `src/quantum_runtime/cli.py`, `src/quantum_runtime/__init__.py`

**`src/quantum_runtime/`:**
- Purpose: Shipped `quantum-runtime` package.
- Contains: CLI entrypoint, runtime orchestration, workspace helpers, intent parsing, canonical `QSpec` models, emitters, diagnostics, reporters, backend integrations, shared error types, and artifact-provenance helpers.
- Key files: `src/quantum_runtime/cli.py`, `src/quantum_runtime/runtime/executor.py`, `src/quantum_runtime/runtime/imports.py`, `src/quantum_runtime/workspace/manager.py`, `src/quantum_runtime/qspec/model.py`, `src/quantum_runtime/artifact_provenance.py`

**`src/quantum_runtime/runtime/`:**
- Purpose: CLI-independent orchestration and read-model layer.
- Contains: `control_plane.py`, `executor.py`, `imports.py`, `compare.py`, `inspect.py`, `export.py`, `pack.py`, `doctor.py`, `policy.py`, `contracts.py`, `observability.py`, `run_manifest.py`, `backend_registry.py`, `backend_list.py`, `ibm_access.py`, `exit_codes.py`, and the barrel `__init__.py`.
- Key files: `src/quantum_runtime/runtime/control_plane.py`, `src/quantum_runtime/runtime/executor.py`, `src/quantum_runtime/runtime/imports.py`, `src/quantum_runtime/runtime/compare.py`, `src/quantum_runtime/runtime/ibm_access.py`

**`src/quantum_runtime/workspace/`:**
- Purpose: File-layout schema, locking, revision metadata, baselines, and trace persistence.
- Contains: `manager.py`, `paths.py`, `manifest.py`, `baseline.py`, `locking.py`, `trace.py`, and the barrel `__init__.py`.
- Key files: `src/quantum_runtime/workspace/manager.py`, `src/quantum_runtime/workspace/paths.py`, `src/quantum_runtime/workspace/manifest.py`, `src/quantum_runtime/workspace/locking.py`

**`src/quantum_runtime/intent/`:**
- Purpose: Ingress parsing and rule-based intent lowering.
- Contains: `parser.py`, `markdown.py`, `structured.py`, `planner.py`, and the barrel `__init__.py`.
- Key files: `src/quantum_runtime/intent/parser.py`, `src/quantum_runtime/intent/planner.py`, `src/quantum_runtime/intent/structured.py`

**`src/quantum_runtime/qspec/`:**
- Purpose: Canonical IR definitions, normalization, semantic hashing, observables, and parameter workflow helpers.
- Contains: `model.py`, `validation.py`, `semantics.py`, `observables.py`, `parameter_workflow.py`, and the barrel `__init__.py`.
- Key files: `src/quantum_runtime/qspec/model.py`, `src/quantum_runtime/qspec/validation.py`, `src/quantum_runtime/qspec/semantics.py`

**`src/quantum_runtime/lowering/`, `src/quantum_runtime/diagnostics/`, and `src/quantum_runtime/backends/`:**
- Purpose: Artifact generation and runtime evidence.
- Contains: Qiskit, QASM3, and Classiq emitters in `src/quantum_runtime/lowering/`; simulation, diagrams, resource estimates, transpile validation, and benchmarks in `src/quantum_runtime/diagnostics/`; optional synthesis backend integration in `src/quantum_runtime/backends/classiq_backend.py`.

**`src/quantum_runtime/reporters/`:**

- Key files: `src/quantum_runtime/reporters/writer.py`, `src/quantum_runtime/reporters/summary.py`

## Key File Locations

**Entry Points:**
- `pyproject.toml`: Package metadata plus the `qrun = "quantum_runtime.cli:main"` console script.
- `src/quantum_runtime/cli.py`: Typer command surface for every shipped CLI command.
- `scripts/dev-bootstrap.sh`: Local install and verification entrypoint.
- `.github/workflows/ci.yml`: Main CI entrypoint.
- `.github/workflows/classiq.yml`: Optional Classiq-specific workflow.

**Configuration:**
- `pyproject.toml`: Dependencies, setuptools discovery, pytest config, and Ruff config.
- `mypy.ini`: Type-check configuration.
- `README.md`: Supported commands, workspace layout, and usage flow.
- `AGENTS.md`: Repository-scoped working instructions for agents.
- `.agents/skills/fluxq-cli/SKILL.md`: FluxQ-specific workflow guidance for agent users.

**Core Logic:**
- `src/quantum_runtime/runtime/control_plane.py`: Resolve, plan, status, show, and schema payloads.
- `src/quantum_runtime/runtime/executor.py`: Exec pipeline, revision reservation, artifact writes, alias promotion, and manifest commit.
- `src/quantum_runtime/runtime/imports.py`: Report or revision reopening, provenance normalization, and replay-integrity checks.
- `src/quantum_runtime/runtime/compare.py`: Semantic, report, backend, and replay-integrity comparison.
- `src/quantum_runtime/runtime/export.py`: Artifact re-export from current or resolved runtime inputs.
- `src/quantum_runtime/runtime/inspect.py`: Compact machine-readable workspace inspection.
- `src/quantum_runtime/runtime/pack.py`: Portable revision bundle assembly and import.
- `src/quantum_runtime/runtime/doctor.py`: Workspace and dependency health checks.
- `src/quantum_runtime/runtime/ibm_access.py`: IBM profile persistence and service construction seam.
- `src/quantum_runtime/artifact_provenance.py`: Canonical artifact path normalization shared by report writing and imports.

**Testing:**
- `tests/test_cli_*.py`: CLI contract and output-mode coverage.
- `tests/test_runtime_*.py`: Runtime module behavior.
- `tests/test_workspace_*.py`: Workspace lifecycle, locking, and baselines.
- `tests/test_pack_bundle.py`, `tests/test_artifact_provenance.py`, `tests/test_release_docs.py`: Delivery, provenance, and release-surface coverage.
- `tests/golden/`: Golden artifacts for emitters and summaries.

## Naming Conventions

**Files:**
- Runtime source files use `snake_case.py`, for example `src/quantum_runtime/runtime/executor.py`, `src/quantum_runtime/workspace/manager.py`, and `src/quantum_runtime/qspec/model.py`.
- CLI and runtime tests use `tests/test_<area>.py`, for example `tests/test_cli_exec.py`, `tests/test_runtime_imports.py`, and `tests/test_workspace_locking.py`.
- Golden snapshots live in `tests/golden/`, for example `tests/golden/qiskit_ghz_main.py` and `tests/golden/qspec_qaoa_maxcut.json`.
- Example intents use `examples/intent-*.md`.
- Release notes use `docs/releases/v*.md`, and planning notes use `docs/plans/YYYY-MM-DD-*.md`.
- Agent skill files follow `SKILL.md` plus helper markdown, as in `.agents/skills/fluxq-cli/SKILL.md` and `.agents/skills/fluxq-cli/quick-reference.md`.

**Directories:**
- Domain packages live under `src/quantum_runtime/<area>/`, for example `runtime/`, `workspace/`, `intent/`, `qspec/`, `lowering/`, `diagnostics/`, `backends/`, and `reporters/`.
- Workspace history follows `<artifact>/history/` conventions under `.quantum/`, for example `reports/history/`, `specs/history/`, `manifests/history/`, `events/history/`, and `trace/history/`.
- Tests stay flat under `tests/` rather than mirroring the source package tree one-to-one.

## Where to Add New Code

**New Feature:**
- Primary code: `src/quantum_runtime/runtime/` for orchestration and read models, plus `src/quantum_runtime/cli.py` if the feature adds or changes a command surface.
- Tests: `tests/test_cli_<feature>.py` for CLI behavior and `tests/test_runtime_<feature>.py` for runtime behavior.

**New Component/Module:**
- Implementation: `src/quantum_runtime/intent/` for ingress parsing or lowering rules, `src/quantum_runtime/qspec/` for IR or semantic rules, `src/quantum_runtime/workspace/` for filesystem or locking logic, and `src/quantum_runtime/lowering/` or `src/quantum_runtime/diagnostics/` for emitters or evidence-producing logic.

**Utilities:**
- Shared helpers: `src/quantum_runtime/artifact_provenance.py`, `src/quantum_runtime/errors.py`, or a narrow helper module inside the package area that owns the behavior.

**Placement Guidance:**
- Put new FluxQ Python code under `src/quantum_runtime/`. No live FluxQ source files were detected in sibling directories under `src/`, so do not place new runtime code under `src/assistant/`, `src/commands/`, `src/components/`, `src/tools/`, or similar empty directories.
- If a feature adds a public runtime API, re-export it through the existing barrels such as `src/quantum_runtime/runtime/__init__.py`, `src/quantum_runtime/workspace/__init__.py`, `src/quantum_runtime/qspec/__init__.py`, or `src/quantum_runtime/reporters/__init__.py`.
- Put host-integration examples in `integrations/` and FluxQ lifecycle guidance for agents in `.agents/skills/fluxq-cli/`.
- Put release-facing docs in `docs/releases/`, deeper design notes in `docs/plans/`, and new sample inputs in `examples/`.
